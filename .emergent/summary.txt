<analysis>original_problem_statement:
The user wants to build a clone of blockchain.com, which will function as a simulated wallet/exchange platform. The platform must look and feel very realistic to the end-user, but all operations (balances, transactions, KYC, etc.) will be controlled and spoofable by an admin.

**PRODUCT REQUIREMENTS:**

1.  **Landing Page:** A professional landing page similar to blockchain.com, containing company information.
2.  **Admin Panel:**
    *   Full control over user accounts (create, view, edit, delete, freeze/unfreeze).
    *   When creating a user, the admin must set:
        *   Personal details (name, email, DOB).
        *   Initial crypto balance (USDC on ETH) and an ETH wallet address.
        *   A simulated transaction history that sums up to the initial balance, generated automatically within a specified date range.
        *   Total unpaid fees, which are automatically distributed among the generated transactions.
        *   A connected app name.
    *   Ability to set two types of account freeze prompts: unusual activity and inactivity.
    *   A KYC queue to approve or decline user submissions.
    *   Ability to manually add deposits, withdrawals, and edit all user data.
3.  **User Flow & Frontend:**
    *   Users see a wallet dashboard (design was provided).
    *   Assets are USDC (ERC-20) and EUR.
    *   If an account is frozen, the user sees a prompt with a button to start the unfreezing process.
    *   **Unusual Activity Flow:** Clicking the button automatically sends an email to the user, guiding them to a KYC page. After KYC approval and a password reset, the account is unfrozen.
    *   **Inactivity Flow:** This flow requires the user to make a real €100 deposit in USDCETH to reactivate the account.
    *   Users must pay the unpaid fees to clear their account for withdrawal, especially when it's marked for closure.
    *   The app should be a Progressive Web App (PWA) with an Install App button.
4.  **KYC Process:**
    *   Users submit ID/passport, a selfie with the document, and proof of address.
    *   The process should be initiated via an automated email link.
5.  **Email System:**
    *   Use Resend for automated emails (KYC, password reset, etc.).
    *   Emails must be fully automated and trigger on specific user actions.

**User's preferred language**: English
